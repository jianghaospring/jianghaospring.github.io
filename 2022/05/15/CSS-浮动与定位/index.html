CSS--浮动与定位

发表于2022-05-15|CSS教程
|字数总计:4.1k|阅读时长:19分钟

学习笔记（CSS）（浮动与定位）

属性（display）

****可以修改元素的类型，让块级元素和内联元素相互转换，还可以隐藏或者显示元素

值 描述
none 此元素不会被显示
block 此元素将显示为块级元素
inline 此元素将显示为内联元素（行内元素）
inline-block 行内块级元素
flex 弹性盒模型

元素隐藏属性对比

属性 描述
display:none 隐藏自己，隐藏后原位置不保留
visibility:hidden 隐藏自己，隐藏后原位置保留
opacity:0 隐藏自己，隐藏后原位置保留
overflow:hidden 溢出部分隐藏

元素分类：

  • 块级元素：div h1~h6… (独占一行，自上而下排列，可以设置宽高)

  • 行内元素(内联元素) a em span. (不独占一行，从左到右排列，不可以设置宽高，内容有多大就显示多大)

  • 行内块级元素 img button… (不独占一行，从左到右排列，可以设置大小)

文档流

文档流是文档中可显示对象在排列时所占用的位置/空间，例如：块元素自上而下摆放，内联元素，从左到右摆放。标准流里面的限制非常多，导致很多页面效果无法实现，在默认的情况下

  • 高矮不齐，底边对齐

  • 空白折叠现象

  • 无论多少个空格、换行、tab，都会折叠为一个空格

  • 如果我们想让img标签之间没有空隙，必须紧密连接，因为回车也会被解析为一个空格

如果我们现在就要并排顶部对齐，那该怎么办呢？办法是：移民！脱离标准流！

使⼀个元素脱离标准文档流有三种方式

  • 浮动

  • 绝对定位

  • 固定定位

浮动

浮动的定义****属性定义元素在哪个方向浮动，任何元素都可以浮动，特别注意：如果下面的元素里有内容，内容不会被浮动覆盖，内容会被挤下去

值 描述
left 元素向左浮动
right 元素向右浮动

浮动的原理

  • 浮动会脱离文档流（相当于在页面上再放一层页面，在新的这层页面上放内容）

  • 浮动只有左右浮动，没有上下浮动

  • 通过float属性定义：取值：left right

  • 浮动元素默认横向摆放

  • 如果横向摆放容器承载不开，会换行

脱离文档流之后，元素相当于在页面上面增加一个浮层来放置内容。此时可以理解为有两层页面，一层是底层的原页面，一层是脱离文档流的上层页面，所以会出现折叠现象

清除浮动

浮动副作用：当元素设置float浮动后，该元素就会脱离文档流并向左 / 向右浮动

  • 浮动元素会造成父元素高度塌陷

  • 后续元素会受到影响

清除浮动：当父元素出现塌陷的时候，对布局是不利的，所以我们必须清除副作用，解决方案有很多种

第一种：父元素设置高度

如果父元素高度塌陷，可以给父元素设置高度，撑开元素本身大小

第二种：受影响的元素增加clear属性 clear: both

值 描述
left 清除左浮动
right 清除右浮动
both（推荐） 同时清除左右浮动

第三种：overflow搭配clear清除浮动（推荐）

如果有父级塌陷，并且同级元素也收到了影响，可以使用overflow清除浮动这种情况下，父布局不能设置高度

第四种：伪对象方式

如果有父级塌陷，并且同级元素也收到了影响，还可以使用伪对象方式处理，为父标签添加伪类after,设置空的内容，并且使用clear:both这种情况下，父布局不能设置高度

浮动实操

定位

定义： ****属性指定了元素的定位类型

值 属性
relative 相对定位
absolute 绝对定位
fixed 固定定位

绝对定位和固定定位会脱离文档流，设置定位之后：可以使用四个方向值进行调整位置：****

相对定位

相对定位不会脱离文档流，实在自己原有的基础上进行上下左右的移动，可以超出屏幕的范围

绝对定位

绝对定位会脱离文档流，并且它的上下左右不是在自己的基础上移动，而是离四边的距离，并且左的优先级大于右，上的优先级大于下，不能超出屏幕，会随着屏幕的滚动而滚动

固定定位

固定定位会脱离文档流，特性和绝对定位一样，但是不会随着屏幕的滚动而滚动

温馨提示

温馨提示：设置定位之后，相对定位和绝对定位他是相对于具有定位的父级元素进行位置调整，如果父级元素不存在定位，则继续向上逐级寻找，直到顶层文档，而固定定位只相对于文档定位

Z-index

z-index属性设置元素的堆叠顺序。拥有更高堆叠顺序的元素总是会处于堆叠顺序较低的元素的前面（只有绝对定位和固定定位有这个属性，因为它们脱离了文档流）
